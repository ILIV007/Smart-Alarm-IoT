# CHAPTER 1: EXECUTIVE SUMMARY

## 1.1 Project Overview

This document presents the comprehensive technical documentation for the **Smart Alarm IoT System**, an advanced security solution that integrates embedded systems, wireless communication, and web technologies to create a modern, intelligent alarm system.

The project implements a dual-microcontroller architecture combining an ATmega328P (Arduino) for sensor management and an ESP8266 for WiFi connectivity, with a Flask-based web server providing real-time monitoring and control capabilities through an elegant, responsive dashboard.

## 1.2 Key Achievements

- **Hybrid Intrusion Detection**: Combines Hall effect sensor and LDR for accurate threat assessment
- **Real-time Monitoring**: 1.5-second refresh rate with live sensor visualization
- **Multi-level Alert System**: Four distinct states with corresponding visual indicators
- **Professional UI/UX**: Modern dark theme with smooth animations and responsive design
- **Robust Communication**: Reliable serial and HTTP protocols with error handling
- **Industrial-grade Logic**: Hysteresis, latching, and cooldown mechanisms for stability

## 1.3 Technical Specifications Summary

| Category | Specification |
|----------|---------------|
| Microcontrollers | ATmega328P + ESP8266 |
| Communication | Serial (9600 baud) + WiFi (HTTP/JSON) |
| Sensors | Hall Effect (analog) + LDR (analog) + Touch (digital) |
| Actuators | 4 LEDs (RGBY) + Active Buzzer |
| Web Framework | Flask (Python 3.8+) |
| Response Time | < 100ms for state changes |
| Power Consumption | ~200mA @ 5V |

# CHAPTER 3: SYSTEM ARCHITECTURE

## 3.1 High-Level Architecture

The system follows a three-tier architecture:

```
┌─────────────────────────────────────────────────────────────┐
│                    PRESENTATION LAYER                       │
│              (Web Dashboard - HTML/CSS/JS)                  │
└──────────────────────┬──────────────────────────────────────┘
                       │ HTTP/JSON
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                     APPLICATION LAYER                       │
│              (Flask Web Server - Python)                    │
└──────────────────────┬──────────────────────────────────────┘
                       │ WiFi (HTTP)
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                       HARDWARE LAYER                        │
│         (ESP8266 + ATmega328P + Sensors + Actuators)        │
└─────────────────────────────────────────────────────────────┘
```

## 3.2 Component Interaction

### Data Flow:
1. **Sensors → ATmega**: Analog/digital readings
2. **ATmega → ESP**: Serial data (sensor values, events)
3. **ESP → Server**: HTTP POST requests (JSON)
4. **Server → Dashboard**: HTTP GET responses (JSON)
5. **Dashboard → Server**: HTTP POST commands (JSON)
6. **Server → ESP**: HTTP GET responses (state updates)
7. **ESP → ATmega**: Serial commands (state changes)
8. **ATmega → Actuators**: LED and buzzer control

## 3.3 Communication Protocols

### Serial Protocol (ATmega ↔ ESP):
- Baud Rate: 9600
- Format: ASCII text with newline delimiter
- Commands: STATE, CMD, DATA, EVENT, CALIB, ACK

### HTTP Protocol (ESP ↔ Server):
- Method: GET/POST
- Content-Type: application/json
- Timeout: 500ms
- Refresh Rate: 2 seconds

## 4.2 Sensor Specifications

### Hall Effect Sensor (A1302/SS49E)
- **Type**: Analog linear
- **Operating Voltage**: 4.5V - 6V
- **Output**: Ratiometric analog (0.5V - 4.5V)
- **Sensitivity**: 1.4 mV/G
- **Range**: ±1000 Gauss
- **Response Time**: < 1 μs

### LDR (GL5528 Photoresistor)
- **Type**: Analog resistive
- **Dark Resistance**: 1-20 MΩ
- **Light Resistance**: 10-20 kΩ @ 10 lux
- **Response Time**: 20-30 ms
- **Spectral Peak**: 540 nm

### Touch Sensor (TTP223)
- **Type**: Capacitive digital
- **Operating Voltage**: 2.0V - 5.5V
- **Output**: Digital (HIGH when touched)
- **Response Time**: ~60 ms
- **Power Consumption**: ~1.5 μA (low power mode)

## 4.3 Actuator Specifications

### Active Buzzer (5V)
- **Operating Voltage**: 3-12V
- **Frequency**: 2300 Hz ± 300 Hz
- **Sound Output**: ≥ 85 dB @ 10 cm
- **Current**: ≤ 30 mA

### LEDs (5mm)
- **Forward Voltage**: 
  - Red: 1.8-2.2V
  - Green: 2.0-2.5V
  - Blue: 3.0-3.5V
  - Yellow: 1.8-2.2V
- **Forward Current**: 20 mA
- **Resistor Value**: 220Ω (calculated for 5V supply)

# CHAPTER 6: INTRUSION DETECTION LOGIC

## 6.1 Detection Algorithm

The intrusion detection system employs a sophisticated algorithm combining sensor filtering, hysteresis, and multi-level threat assessment.

### Step 1: Sensor Filtering
```cpp
// Moving average filter to reduce noise
int hallFiltered = getHallFiltered();  // 5-sample average
int ldrFiltered = getLdrFiltered();    // 5-sample average
```

### Step 2: Hysteresis Implementation
```cpp
// Prevent oscillation with hysteresis bands
bool hallDanger;
if (lastHallDanger) {
  // Already in danger: need wider band to exit
  hallDanger = (hall >= HALL_SAFE_MIN && hall <= HALL_SAFE_MAX);
} else {
  // Not in danger: need narrower band to enter
  hallDanger = (hall >= HALL_DANGER_MIN && hall <= HALL_DANGER_MAX);
}
```

### Step 3: Threat Assessment
```cpp
if (hallDanger && ldrDanger) {
  // CRITICAL: Both sensors triggered
  state = TRIGGERED;
  sirenActive = true;
} 
else if (hallDanger || ldrDanger) {
  // WARNING: One sensor triggered
  state = SUSPICIOUS;
  sirenActive = false;
} 
else {
  // SAFE: No sensors triggered
  state = ARMED;
  sirenActive = false;
}
```

## 6.2 Hysteresis Parameters

| Parameter | Value | Purpose |
|-----------|-------|---------|
| HALL_DANGER_MIN | 450 | Entry threshold (lower bound) |
| HALL_DANGER_MAX | 550 | Entry threshold (upper bound) |
| HALL_SAFE_MIN | 435 | Exit threshold (lower bound) |
| HALL_SAFE_MAX | 565 | Exit threshold (upper bound) |
| LDR_DANGER_THRESHOLD | 40 | Danger entry threshold |
| LDR_SAFE_THRESHOLD | 60 | Danger exit threshold |

## 6.3 Latch Mechanism

Once the system enters the TRIGGERED state, it remains locked until manual intervention:

```cpp
if (sysState == TRIGGERED) {
  return; // Ignore all sensor changes
}
```

This prevents the alarm from automatically resetting when the intrusion source is removed, ensuring the user is aware of the security breach.

## 6.4 Cooldown Protection

After touch activation, a 5-second cooldown prevents false triggers:

```cpp
if (ev == "TOUCH_ARM_SHORT") {
  sysState = ARMED;
  alarmCooldown = true;
  cooldownEnd = millis() + 5000; // 5 seconds
}
```

During this period, the `checkIntrusion()` function is bypassed, allowing the user to move away from sensors without triggering an alarm.

## 9.2 Hardware Assembly

### Step 1: Connect Hall Sensor
```
Hall Sensor VCC → Arduino 5V
Hall Sensor GND → Arduino GND
Hall Sensor OUT → Arduino A0
```

### Step 2: Connect LDR
```
5V → LDR → Arduino A1 → 10KΩ → GND
```

### Step 3: Connect Touch Sensor
```
Touch Sensor VCC → Arduino 5V
Touch Sensor GND → Arduino GND
Touch Sensor OUT → Arduino D4
```

### Step 4: Connect Buzzer
```
Buzzer Positive (+) → Arduino D5
Buzzer Negative (-) → Arduino GND
```

### Step 5: Connect LEDs
```
Red LED Anode → 220Ω → Arduino D9
Green LED Anode → 220Ω → Arduino D10
Blue LED Anode → 220Ω → Arduino D11
Yellow LED Anode → 220Ω → Arduino D12
All LED Cathodes → Arduino GND
```

### Step 6: Connect ESP8266
```
ESP TX → Arduino RX
ESP RX → Arduino TX (via voltage divider if needed)
ESP VCC → 3.3V or 5V (depending on module)
ESP GND → Arduino GND
```

# CHAPTER 10: USER MANUAL

## 10.1 Initial Setup

### First Boot Sequence
1. Power on the system
2. LEDs will cycle: Red → Green → Blue → Yellow (2.4 seconds)
3. System calibrates sensors automatically (2 seconds)
4. Blue LED indicates DISARMED state

### Verify System Status
1. Open web dashboard
2. Check all components show "Online"
3. Verify sensor readings are reasonable:
   - Hall: 450-550 (normal range)
   - LDR: 400-600 (ambient light)

## 10.2 Daily Operation

### Arming the System

**Method 1: Touch Sensor**
- Tap touch sensor briefly (< 0.5 seconds)
- LED turns GREEN
- Dashboard shows "ACTIVE" status

**Method 2: Web Dashboard**
- Click on the central orb
- System arms immediately
- LED turns GREEN

### Disarming the System

**Method 1: Touch Sensor**
- Tap touch sensor briefly
- System disarms (if not in TRIGGERED state)
- LED turns BLUE

**Method 2: Web Dashboard**
- Click on the central orb
- Enter password (default: 0000)
- Click "Confirm"
- LED turns BLUE

### Manual Alarm (Panic Mode)
- Press and hold touch sensor for 2 seconds
- LED turns RED immediately
- Buzzer starts pulsing
- Dashboard shows "ALERT" status

## 10.3 Understanding LED States

| Color | State | Meaning | Buzzer | Action Required |
|-------|-------|---------|--------|-----------------|
| 🔵 Blue | DISARMED | System inactive | OFF | None |
| 🟢 Green | ARMED | System active | OFF | Monitor sensors |
| 🟡 Yellow | SUSPICIOUS | Potential threat | OFF | Investigate |
| 🔴 Red | TRIGGERED | Confirmed intrusion | ON | Reset system |

## 10.4 Sensor Behavior

### Hall Sensor (Door/Window)
- **Normal**: 450-550 (magnet near)
- **Danger**: 450-550 (magnet removed)
- **Safe**: < 435 or > 565

### LDR Sensor (Light)
- **Normal**: 400-600 (ambient light)
- **Danger**: < 40 (sudden darkness)
- **Safe**: > 60

# CHAPTER 12: TROUBLESHOOTING

## 12.1 Common Issues and Solutions

### Issue 1: LEDs Not Lighting Up

**Symptoms**:
- LEDs remain dark
- Wrong colors displayed

**Solutions**:
1. Check LED polarity (long leg = anode)
2. Verify 220Ω resistors are connected
3. Check pin connections match code
4. Run LED test in setup

---

### Issue 2: Buzzer Not Sounding

**Symptoms**:
- No sound when triggered

**Solutions**:
1. Verify buzzer is active type
2. Check polarity
3. Test with simple sketch
4. Replace buzzer if defective

---

### Issue 3: Hall Sensor Not Detecting

**Symptoms**:
- Value doesn't change with magnet

**Solutions**:
1. Verify sensor is analog type
2. Check wiring
3. Test with multimeter
4. Try stronger magnet

---

### Issue 4: LDR Not Responding

**Symptoms**:
- Value stays constant

**Solutions**:
1. Verify voltage divider circuit
2. Check 10KΩ resistor
3. Test with multimeter
4. Replace LDR

---

### Issue 5: WiFi Not Connecting

**Symptoms**:
- "WiFi: Failed" in Serial Monitor

**Solutions**:
1. Verify SSID and password (case-sensitive)
2. Check WiFi is 2.4GHz
3. Move ESP closer to router
4. Check router MAC filtering

---

### Issue 6: Dashboard Shows "Offline"

**Symptoms**:
- Device shows offline

**Solutions**:
1. Verify server IP in ESP code
2. Check firewall settings
3. Ensure server is running
4. Check ESP Serial Monitor

---

### Issue 7: Wrong LED Colors

**Symptoms**:
- Colors don't match states

**Solutions**:
1. Check pin assignments in code
2. Verify LED colors match pins
3. Swap pin numbers if needed

---

### Issue 8: False Alarms

**Symptoms**:
- System triggers without intrusion

**Solutions**:
1. Adjust hysteresis thresholds
2. Check for EMI
3. Verify magnet mounting
4. Increase cooldown

---

### Issue 9: System Stuck in TRIGGERED

**Symptoms**:
- Won't exit alarm state

**Solutions**:
1. Use web dashboard to disarm
2. Press touch sensor briefly
3. Power cycle system
4. Check Serial Monitor
