## 🎯 Overview

This project implements a sophisticated smart alarm system that combines embedded systems with web technologies to create a comprehensive security solution. The system uses a dual-microcontroller architecture (ATmega328P + ESP8266) with a Flask-based web server for real-time monitoring and control.

### What Makes This Project Special?

- **Hybrid Intrusion Detection**: Combines Hall effect sensor and LDR (Light Dependent Resistor) for accurate intrusion detection
- **Real-time Web Dashboard**: Beautiful, responsive web interface with live sensor data visualization
- **Multi-level Alerts**: Four distinct states (Disarmed, Armed, Suspicious, Triggered) with corresponding LED indicators
- **Touch-based Control**: Intuitive touch sensor for local control without needing the web interface
- **Siren System**: Ambulance-style pulsing alarm when intrusion is detected
- **Event Logging**: Complete history of all system events with CSV export capability
- **Professional UI/UX**: Modern dark theme with smooth animations and responsive design

---

## ✨ Key Features

### 🎛️ Hardware Features
- **4 Status LEDs**: Red (Triggered), Green (Armed), Blue (Disarmed), Yellow (Suspicious)
- **Hall Effect Sensor**: Detects door/window opening with magnetic field changes
- **LDR Sensor**: Detects light changes indicating potential intrusion
- **Touch Sensor**: Capacitive touch for local system control
- **Active Buzzer**: Audible alarm with pulsing pattern
- **Dual Microcontroller**: ATmega328P for sensor control + ESP8266 for WiFi connectivity

### 💻 Software Features
- **Real-time Monitoring**: Live sensor data with 1.5-second refresh rate
- **Interactive Charts**: Dynamic line charts showing sensor history
- **Color-coded Alerts**: Charts change color based on threat level
- **Event History**: Timestamped log of all system events
- **CSV Export**: Download event history for analysis
- **Responsive Design**: Works perfectly on desktop, tablet, and mobile
- **Dark/Light Themes**: User-selectable interface themes
- **Audio Alerts**: Browser-based alarm sounds
- **Favicon Integration**: Custom lock icon in browser tab

### 🔐 Security Features
- **Password Protection**: Web dashboard requires authentication (default: 0000)
- **Latch Logic**: Once triggered, system stays in alarm state until manually reset
- **Cooldown Period**: 5-second protection after touch activation to prevent false triggers
- **Hysteresis**: Advanced filtering to prevent sensor noise from causing false alarms
- **Dual Verification**: Both Hall and LDR must trigger for full alarm (configurable)

### Communication Protocol

**ATmega → ESP (Serial)**
```
DATA|H:{hall_value},L:{ldr_value},T:{touch_state},C:{calibrated}
EVENT:TOUCH_ARM_SHORT
EVENT:TOUCH_ALARM_3S
CALIB:{hall_base},{ldr_base}
ACK:{command}
```

**ESP → ATmega (Serial)**
```
STATE:DISARMED|ARMED|SUSPICIOUS|TRIGGERED
CMD:BUZZER_ON|BUZZER_OFF
CMD:RGB:{r},{g},{b},{y}
CMD:CALIBRATE
```

**ESP ↔ Server (HTTP/JSON)**
```json

## 🔧 Hardware Requirements

### Core Components
- **Arduino Uno** (or compatible ATmega328P board)
- **ESP8266 WiFi Module** (ESP-01, NodeMCU, or Wemos D1 Mini)
- **Hall Effect Sensor** (analog output, e.g., A1302, SS49E)
- **LDR (Photoresistor)** with 10KΩ pull-down resistor
- **Capacitive Touch Sensor** (TTP223 or similar)
- **Active Buzzer** (5V)
- **4x LEDs** (Red, Green, Blue, Yellow) with 220Ω resistors
- **Breadboard and jumper wires**
- **5V Power supply** (USB or adapter)

### Optional Components
- **Magnet** (for Hall sensor testing)
- **Enclosure/Case** for professional installation
- **PCB** for permanent installation

### Pin Connections

**ATmega328P Pin Mapping:**
```
A0 - Hall Effect Sensor (analog input)
A1 - LDR Sensor (analog input, with 10KΩ to GND)
D4 - Touch Sensor (digital input)
D5 - Active Buzzer (digital output)
D9 - Red LED (PWM output)
D10 - Green LED (PWM output)
D11 - Blue LED (PWM output)
D12 - Yellow LED (digital output)
GND - Common ground for all components
5V - Power supply for sensors and LEDs
```

**ESP8266 Pin Mapping:**
```
TX - ATmega RX (via voltage divider if needed)
RX - ATmega TX
VCC - 3.3V or 5V (depending on module)
GND - Common ground
```

## 🚀 Installation & Setup

### 1. Hardware Assembly

1. **Connect Hall Sensor:**
   - VCC → 5V
   - GND → GND
   - OUT → A0

2. **Connect LDR:**
   - One leg → 5V
   - Other leg → A1 AND 10KΩ resistor → GND

3. **Connect Touch Sensor:**
   - VCC → 5V
   - GND → GND
   - OUT → D4

4. **Connect Buzzer:**
   - Positive (+) → D5
   - Negative (-) → GND

5. **Connect LEDs (with 220Ω resistors):**
   - Red LED anode → D9
   - Green LED anode → D10
   - Blue LED anode → D11
   - Yellow LED anode → D12
   - All cathodes → GND

6. **Connect ESP8266:**
   - TX → ATmega RX
   - RX → ATmega TX
   - VCC → 3.3V or 5V
   - GND → GND

### 2. Software Installation

#### Prerequisites
- **Arduino IDE** (version 1.8.x or higher)
- **Python 3.8+** with pip
- **Git** (optional, for cloning)

#### Install Arduino Libraries
Open Arduino IDE and install these libraries via Library Manager:
- `ESP8266WiFi` (for ESP8266)
- `ESP8266HTTPClient` (for ESP8266)
- `ArduinoJson` (version 6.x, for both boards)

#### Upload Firmware

**Step 1: Upload ATmega Code**
1. Open `Firmware/SmartAlarm_ATmega.ino` in Arduino IDE
2. Select board: `Arduino Uno`
3. Select correct COM port
4. Click Upload
5. Wait for "Done uploading" message

**Step 2: Upload ESP8266 Code**
1. Open `Firmware/SmartAlarm_ESP8266.ino` in Arduino IDE
2. Select board: `Generic ESP8266 Module` (or your specific board)
3. **Edit WiFi credentials** (lines 10-12):
   ```cpp
   const char* WIFI_SSID = "YourWiFiName";
   const char* WIFI_PASS = "YourWiFiPassword";
   const char* SERVER_HOST = "192.168.1.50"; // Your server IP
   ```
4. Select correct COM port
5. Click Upload
6. Wait for "Done uploading" message

#### Install Server Dependencies

```bash
# Navigate to server directory
cd Server

# Create virtual environment (recommended)
python -m venv venv

# Activate virtual environment
# On Windows:
venv\Scripts\activate
# On macOS/Linux:
source venv/bin/activate

# Install dependencies
pip install flask
```

### 3. Configuration

#### Edit Server IP
In `SmartAlarm_ESP8266.ino`, update line 12:
```cpp
const char* SERVER_HOST = "YOUR_SERVER_IP";
```

To find your server IP:
- **Windows**: Open CMD and run `ipconfig`
- **macOS/Linux**: Open Terminal and run `ifconfig` or `ip addr`

#### Edit WiFi Credentials
In `SmartAlarm_ESP8266.ino`, update lines 10-11:
```cpp
const char* WIFI_SSID = "YourWiFiName";
const char* WIFI_PASS = "YourWiFiPassword";
```

#### Change Admin Password
In `app.py`, update line 7:
```python
ADMIN_PASS = "0000"  # Change to your desired password
```

### 4. Run the Server

```bash
# Make sure you're in the Server directory
cd Server

# Run the Flask server
python app.py
```

You should see:
```
🚀 Server running: http://192.168.1.50:5000 | Pass: 0000
 * Serving Flask app 'app'
 * Debug mode: off
 * Running on all addresses (0.0.0.0)
 * Running on http://127.0.0.1:5000
 * Running on http://192.168.1.50:5000
```

### 5. Access the Dashboard

1. Open a web browser
2. Navigate to: `http://YOUR_SERVER_IP:5000`
3. Enter password: `0000` (or your custom password)
4. You should see the dashboard with live sensor data

## 📖 Usage Guide

### Initial Setup

1. **Power on the system**
   - LEDs will cycle through Red → Green → Blue → Yellow (testing)
   - System calibrates sensors automatically
   - Blue LED indicates DISARMED state

2. **Check Web Dashboard**
   - Open browser and navigate to server IP
   - Verify all sensors show "Online" status
   - Check sensor readings are reasonable

3. **Test Sensors**
   - **Hall Sensor**: Move magnet near/far from sensor
   - **LDR**: Cover/uncover with hand or shine light
   - Verify values change in dashboard

### Daily Operation

#### Arming the System

**Method 1: Touch Sensor**
- Tap touch sensor briefly (< 0.5 seconds)
- LED turns GREEN
- Dashboard shows "ACTIVE" status

**Method 2: Web Dashboard**
- Click on the central orb/planet
- System arms immediately
- LED turns GREEN

#### Disarming the System

**Method 1: Touch Sensor**
- Tap touch sensor briefly
- System disarms (if not in TRIGGERED state)
- LED turns BLUE

**Method 2: Web Dashboard**
- Click on the central orb
- Enter password (default: 0000)
- Click "Confirm"
- LED turns BLUE

#### Manual Alarm (Panic Mode)

- Press and hold touch sensor for 2 seconds
- LED turns RED immediately
- Buzzer starts pulsing
- Dashboard shows "ALERT" status
- Event logged as "MANUAL" alarm

### Understanding LED States

| LED Color | State | Meaning | Buzzer |
|-----------|-------|---------|--------|
| 🔵 Blue | DISARMED | System inactive, monitoring disabled | OFF |
| 🟢 Green | ARMED | System active, monitoring enabled | OFF |
| 🟡 Yellow | SUSPICIOUS | One sensor triggered, potential threat | OFF |
| 🔴 Red | TRIGGERED | Both sensors triggered, confirmed intrusion | ON (pulsing) |

### Sensor Behavior

#### Hall Sensor (Door/Window)
- **Normal range**: 450-550 (when magnet is near)
- **Danger range**: 450-550 (when magnet is removed)
- **Safe range**: < 435 or > 565

#### LDR Sensor (Light)
- **Normal**: 400-600 (ambient light)
- **Danger**: < 40 (sudden darkness, e.g., door opening in dark room)
- **Safe**: > 60

### Event History

- View all events in the "Event History" section
- Each event shows:
  - Timestamp
  - Event type (ARMED, DISARMED, ALARM, MANUAL)
  - Severity level (Critical, Warning, Info)
  - Sensor data at time of event

### CSV Export

1. Click "Export CSV" button
2. File downloads automatically
3. Open in Excel, Google Sheets, or any spreadsheet software
4. Analyze event patterns and timing

## 🔍 Intrusion Detection Logic

### State Machine

```
                    ┌─────────────┐
         ┌─────────│  DISARMED   │◄─────────┐
         │         └─────────────┘          │
         │                │                 │
         │           Touch/Website          │
         │                │                 │
         │                ▼                 │
         │         ┌─────────────┐          │
         │         │   ARMED     │          │
         │         └─────────────┘          │
         │                │                 │
         │         One sensor               │
         │         triggered                │
         │                │                 │
         │                ▼                 │
         │         ┌─────────────┐          │
         │         │ SUSPICIOUS  │          │
         │         └─────────────┘          │
         │                │                 │
         │         Both sensors             │
         │         triggered                │
         │                │                 │
         │                ▼                 │
         │         ┌─────────────┐          │
         └─────────│  TRIGGERED  │──────────┘
                   └─────────────┘
                   (Latch state,
                   requires manual
                   reset)
```

### Detection Algorithm

#### Step 1: Sensor Filtering
```cpp
// Moving average filter (5 samples)
int hallFiltered = getHallFiltered();
int ldrFiltered = getLdrFiltered();
```

#### Step 2: Hysteresis Check
```cpp
// Prevent oscillation with hysteresis bands
bool hallDanger;
if (lastHallDanger) {
  // Already in danger: need to exit safe zone
  hallDanger = (hall >= HALL_SAFE_MIN && hall <= HALL_SAFE_MAX);
} else {
  // Not in danger: need to enter danger zone
  hallDanger = (hall >= HALL_DANGER_MIN && hall <= HALL_DANGER_MAX);
}
```

#### Step 3: Threat Assessment
```cpp
// Calculate threat level
if (hallDanger && ldrDanger) {
  // CRITICAL: Both sensors triggered
  state = TRIGGERED;
  sirenActive = true;
} 
else if (hallDanger || ldrDanger) {
  // WARNING: One sensor triggered
  state = SUSPICIOUS;
  sirenActive = false;
} 
else {
  // SAFE: No sensors triggered
  state = ARMED;
  sirenActive = false;
}
```

#### Step 4: Latch Logic
```cpp
// Once TRIGGERED, stay in alarm until manual reset
if (sysState == TRIGGERED) {
  return; // Ignore sensor changes
}
```

### Cooldown Mechanism

After touch activation, a 5-second cooldown prevents false triggers:

```cpp
if (ev == "TOUCH_ARM_SHORT") {
  sysState = ARMED;
  alarmCooldown = true;
  cooldownEnd = millis() + 5000; // 5 seconds
  
  // During cooldown, checkIntrusion() is skipped
}
```

## 🐛 Troubleshooting

### Issue: LEDs Not Lighting Up

**Symptoms:**
- LEDs remain dark or show wrong colors

**Solutions:**
1. Check LED polarity (long leg = anode = positive)
2. Verify 220Ω resistors are connected
3. Check pin connections match code (D9, D10, D11, D12)
4. Run LED test in setup (should cycle through colors on boot)

---

### Issue: Buzzer Not Sounding

**Symptoms:**
- No sound when system triggers

**Solutions:**
1. Verify buzzer is **active** type (not passive)
2. Check polarity (positive leg to D5)
3. Test with simple sketch:
   ```cpp
   void setup() {
     pinMode(5, OUTPUT);
   }
   void loop() {
     digitalWrite(5, HIGH);
     delay(1000);
     digitalWrite(5, LOW);
     delay(1000);
   }
   ```

---

### Issue: Hall Sensor Not Detecting

**Symptoms:**
- Hall value doesn't change with magnet

**Solutions:**
1. Verify sensor is **analog** type (not digital)
2. Check wiring: VCC→5V, GND→GND, OUT→A0
3. Test with multimeter (should read 0-5V)
4. Try different magnet (stronger neodymium recommended)
5. Check Serial Monitor for raw values

---

### Issue: LDR Not Responding

**Symptoms:**
- LDR value stays constant

**Solutions:**
1. Verify voltage divider circuit:
   ```
   5V → LDR → A1 → 10KΩ → GND
   ```
2. Check 10KΩ resistor value
3. Test with multimeter (resistance should change with light)
4. Try different LDR (GL5528 recommended)

---

### Issue: Touch Sensor Not Working

**Symptoms:**
- Touch doesn't trigger events

**Solutions:**
1. Check wiring: VCC→5V, GND→GND, OUT→D4
2. Verify sensor type (TTP223 or similar)
3. Test with simple sketch:
   ```cpp
   void setup() {
     Serial.begin(9600);
     pinMode(4, INPUT);
   }
   void loop() {
     Serial.println(digitalRead(4));
     delay(100);
   }
   ```
4. Adjust touch sensitivity (if module has potentiometer)

---

### Issue: WiFi Not Connecting

**Symptoms:**
- ESP8266 shows "WiFi: Failed" in Serial Monitor

**Solutions:**
1. Verify SSID and password in code (case-sensitive!)
2. Check WiFi is 2.4GHz (ESP8266 doesn't support 5GHz)
3. Move ESP closer to router
4. Check router MAC filtering (add ESP MAC address)
5. Try different WiFi network

---

### Issue: Dashboard Shows "Offline"

**Symptoms:**
- Web dashboard shows device offline

**Solutions:**
1. Verify server IP in ESP code matches actual server IP
2. Check firewall settings (allow port 5000)
3. Ensure server is running (`python app.py`)
4. Check ESP Serial Monitor for HTTP errors
5. Verify both devices are on same network

---

### Issue: Wrong LED Colors

**Symptoms:**
- Red shows as Blue, Green shows as Yellow, etc.

**Solutions:**
1. Check pin assignments in code match physical wiring
2. Verify LED colors match pin labels
3. Swap pin numbers in code if needed:
   ```cpp
   const int PIN_RED = 11;    // Swap with BLUE
   const int PIN_BLUE = 9;    // Swap with RED
   ```

---

### Issue: False Alarms

**Symptoms:**
- System triggers without actual intrusion

**Solutions:**
1. Adjust hysteresis thresholds in ESP code:
   ```cpp
   #define HALL_DANGER_MIN 460  // Increase to narrow danger zone
   #define HALL_DANGER_MAX 540  // Decrease to narrow danger zone
   ```
2. Check for electromagnetic interference (motors, transformers)
3. Verify magnet is securely mounted
4. Increase `ALARM_COOLDOWN` value
5. Check LDR for sudden light changes (shadows, clouds)

---

### Issue: System Stuck in TRIGGERED State

**Symptoms:**
- System won't exit alarm state

**Solutions:**
1. Use web dashboard to disarm (enter password)
2. Press touch sensor briefly to reset
3. Power cycle the system
4. Check Serial Monitor for error messages
5. Verify `syncStatus()` is working in ESP code

---

### Issue: Serial Communication Errors

**Symptoms:**
- Garbled data between ATmega and ESP

**Solutions:**
1. Verify baud rate matches (9600 in both sketches)
2. Check TX/RX connections (cross them: TX→RX, RX→TX)
3. Add voltage divider if ESP is 3.3V and ATmega is 5V:
   ```
   ATmega TX → 1KΩ → ESP RX → 2KΩ → GND
   ```
4. Use SoftwareSerial for debugging
5. Check for ground loops (connect all GND together)

## 🔒 Security Considerations

### Current Security Measures
- Password protection for web dashboard
- Local network only (not exposed to internet)
- No sensitive data stored in plain text
- Event logging for audit trail

### Recommended Enhancements
1. **HTTPS**: Use SSL/TLS for encrypted communication
2. **Strong Passwords**: Enforce complex password requirements
3. **Rate Limiting**: Prevent brute-force attacks on login
4. **Session Management**: Implement secure session tokens
5. **Input Validation**: Sanitize all user inputs
6. **Network Segmentation**: Place IoT devices on separate VLAN
7. **Regular Updates**: Keep all software components updated
8. **Two-Factor Authentication**: Add 2FA for web access

### Known Limitations
- Default password (0000) should be changed immediately
- No encryption for serial communication between microcontrollers
- WiFi credentials stored in plain text in firmware
- No intrusion detection for the system itself

### چه چیزی این پروژه را خاص می‌کند؟

- **سیستم آژیر**: آژیر ضربانی به سبک آمبولانس هنگام تشخیص نفوذ
- **ثبت رویدادها**: تاریخچه کامل تمام رویدادهای سیستم با قابلیت خروجی CSV

## 🔧 سخت‌افزار مورد نیاز

### اجزای اصلی
- **Arduino Uno** (یا برد سازگار ATmega328P)
- **ماژول WiFi ESP8266** (ESP-01, NodeMCU, یا Wemos D1 Mini)
- **سنسور اثر هال** (خروجی آنالوگ، مانند A1302, SS49E)
- **LDR (فتومقاومت)** با مقاومت pull-down 10KΩ
- **سنسور لمسی خازنی** (TTP223 یا مشابه)
- **بازر فعال** (5V)
- **4 عدد LED** (قرمز، سبز، آبی، زرد) با مقاومت‌های 220Ω
- **برد بورد و سیم‌های جامپر**
- **منبع تغذیه 5V** (USB یا آداپتور)

### اجزای اختیاری
- **آهنربا** (برای تست سنسور هال)
- **محفظه/کیس** برای نصب حرفه‌ای
- **PCB** برای نصب دائمی

### اتصالات پین

**نگاشت پین ATmega328P:**
```
A0 - سنسور اثر هال (ورودی آنالوگ)
A1 - سنسور LDR (ورودی آنالوگ، با 10KΩ به GND)
D4 - سنسور لمسی (ورودی دیجیتال)
D5 - بازر فعال (خروجی دیجیتال)
D9 - LED قرمز (خروجی PWM)
D10 - LED سبز (خروجی PWM)
D11 - LED آبی (خروجی PWM)
D12 - LED زرد (خروجی دیجیتال)
GND - زمین مشترک برای تمام اجزا
5V - منبع تغذیه برای سنسورها و LEDها
```

**نگاشت پین ESP8266:**
```
TX - ATmega RX (در صورت نیاز از طریق تقسیم‌کننده ولتاژ)
RX - ATmega TX
VCC - 3.3V یا 5V (بسته به ماژول)
GND - زمین مشترک
```

## 🚀 نصب و راه‌اندازی

### 1. مونتاژ سخت‌افزار

1. **اتصال سنسور هال:**
   - VCC → 5V
   - GND → GND
   - OUT → A0

2. **اتصال LDR:**
   - یک پایه → 5V
   - پایه دیگر → A1 و مقاومت 10KΩ → GND

3. **اتصال سنسور لمسی:**
   - VCC → 5V
   - GND → GND
   - OUT → D4

4. **اتصال بازر:**
   - مثبت (+) → D5
   - منفی (-) → GND

5. **اتصال LEDها (با مقاومت‌های 220Ω):**
   - آند LED قرمز → D9
   - آند LED سبز → D10
   - آند LED آبی → D11
   - آند LED زرد → D12
   - تمام کاتدها → GND

6. **اتصال ESP8266:**
   - TX → ATmega RX
   - RX → ATmega TX
   - VCC → 3.3V یا 5V
   - GND → GND

### 2. نصب نرم‌افزار

#### پیش‌نیازها
- **Arduino IDE** (نسخه 1.8.x یا بالاتر)
- **Python 3.8+** با pip
- **Git** (اختیاری، برای کلون کردن)

## 📖 راهنمای استفاده

### راه‌اندازی اولیه

1. **سیستم را روشن کنید**
   - LEDها به ترتیب قرمز → سبز → آبی → زرد چشمک می‌زنند (تست)
   - سیستم سنسورها را به طور خودکار کالیبره می‌کند
   - LED آبی نشان‌دهنده حالت غیرفعال است

2. **داشبورد وب را بررسی کنید**
   - مرورگر را باز کنید و به IP سرور بروید
   - تأیید کنید تمام سنسورها وضعیت "آنلاین" را نشان می‌دهند
   - بررسی کنید خوانش‌های سنسور منطقی هستند

3. **سنسورها را تست کنید**
   - **سنسور هال**: آهنربا را نزدیک/دور از سنسور حرکت دهید
   - **LDR**: با دست بپوشانید/باز کنید یا نور بتابانید
   - تأیید کنید مقادیر در داشبورد تغییر می‌کنند

### عملیات روزانه

#### فعال‌سازی سیستم

**روش 1: سنسور لمسی**
- سنسور لمسی را به طور کوتاه ضربه بزنید (< 0.5 ثانیه)
- LED سبز می‌شود
- داشبورد وضعیت "ACTIVE" را نشان می‌دهد

**روش 2: داشبورد وب**
- روی کره/سیاره مرکزی کلیک کنید
- سیستم بلافاصله فعال می‌شود
- LED سبز می‌شود

#### غیرفعال‌سازی سیستم

**روش 1: سنسور لمسی**
- سنسور لمسی را به طور کوتاه ضربه بزنید
- سیستم غیرفعال می‌شود (اگر در حالت فعال‌شده نباشد)
- LED آبی می‌شود

**روش 2: داشبورد وب**
- روی کره مرکزی کلیک کنید
- رمز عبور را وارد کنید (پیش‌فرض: 0000)
- روی "تأیید" کلیک کنید
- LED آبی می‌شود

#### هشدار دستی (حالت وحشت)

- سنسور لمسی را به مدت 2 ثانیه نگه دارید
- LED بلافاصله قرمز می‌شود
- بازر شروع به ضربان زدن می‌کند
- داشبورد وضعیت "ALERT" را نشان می‌دهد
- رویداد به عنوان هشدار "MANUAL" ثبت می‌شود

### درک حالت‌های LED

| رنگ LED | حالت | معنی | بازر |
|---------|------|------|------|
| 🔵 آبی | غیرفعال | سیستم غیرفعال، نظارت غیرفعال | خاموش |
| 🟢 سبز | فعال | سیستم فعال، نظارت فعال | خاموش |
| 🟡 زرد | مشکوک | یک سنسور فعال شده، تهدید احتمالی | خاموش |
| 🔴 قرمز | فعال‌شده | هر دو سنسور فعال شده، نفوذ تأیید شده | روشن (ضربانی) |

### رفتار سنسورها

#### سنسور هال (در/پنجره)
- **محدوده عادی**: 450-550 (وقتی آهنربا نزدیک است)
- **محدوده خطر**: 450-550 (وقتی آهنربا حذف شده)
- **محدوده امن**: < 435 یا > 565

#### سنسور LDR (نور)
- **عادی**: 400-600 (نور محیط)
- **خطر**: < 40 (تاریکی ناگهانی، مثلاً باز شدن در در اتاق تاریک)
- **امن**: > 60

### تاریخچه رویدادها

- تمام رویدادها را در بخش "تاریخچه رویدادها" مشاهده کنید
- هر رویداد نشان می‌دهد:
  - زمان‌بندی
  - نوع رویداد (فعال‌شده، غیرفعال‌شده، هشدار، دستی)
  - سطح شدت (بحرانی، هشدار، اطلاعاتی)
  - داده‌های سنسور در زمان رویداد

### خروجی CSV

1. روی دکمه "خروجی CSV" کلیک کنید
2. فایل به طور خودکار دانلود می‌شود
3. در Excel، Google Sheets یا هر نرم‌افزار صفحه‌گسترده باز کنید
4. الگوها و زمان‌بندی رویدادها را تحلیل کنید

### الگوریتم تشخیص

#### مرحله 1: فیلتر کردن سنسور
```cpp
// فیلتر میانگین متحرک (5 نمونه)
int hallFiltered = getHallFiltered();
int ldrFiltered = getLdrFiltered();
```

#### مرحله 2: بررسی هیسترزیس
```cpp
// جلوگیری از نوسان با باندهای هیسترزیس
bool hallDanger;
if (lastHallDanger) {
  // قبلاً در خطر: باید از منطقه امن خارج شود
  hallDanger = (hall >= HALL_SAFE_MIN && hall <= HALL_SAFE_MAX);
} else {
  // در خطر نیست: باید وارد منطقه خطر شود
  hallDanger = (hall >= HALL_DANGER_MIN && hall <= HALL_DANGER_MAX);
}
```

#### مرحله 3: ارزیابی تهدید
```cpp
// محاسبه سطح تهدید
if (hallDanger && ldrDanger) {
  // بحرانی: هر دو سنسور فعال شده
  state = TRIGGERED;
  sirenActive = true;
} 
else if (hallDanger || ldrDanger) {
  // هشدار: یک سنسور فعال شده
  state = SUSPICIOUS;
  sirenActive = false;
} 
else {
  // امن: هیچ سنسوری فعال نشده
  state = ARMED;
  sirenActive = false;
}
```

#### مرحله 4: منطق قفل‌شونده
```cpp
// پس از فعال‌شدن، تا بازنشانی دستی در هشدار می‌ماند
if (sysState == TRIGGERED) {
  return; // تغییرات سنسور را نادیده بگیر
}
```

### مکانیزم استراحت

پس از فعال‌سازی لمسی، استراحت 5 ثانیه‌ای از تریگرهای کاذب جلوگیری می‌کند:

```cpp
if (ev == "TOUCH_ARM_SHORT") {
  sysState = ARMED;
  alarmCooldown = true;
  cooldownEnd = millis() + 5000; // 5 ثانیه
  
  // در طول استراحت، checkIntrusion() رد می‌شود

## 🐛 عیب‌یابی

### مشکل: LEDها روشن نمی‌شوند

**علائم:**
- LEDها خاموش می‌مانند یا رنگ‌های اشتباه نشان می‌دهند

**راه‌حل‌ها:**
1. قطبیت LED را بررسی کنید (پایه بلند = آند = مثبت)
2. تأیید کنید مقاومت‌های 220Ω متصل هستند
3. بررسی کنید اتصالات پین با کد مطابقت دارند (D9, D10, D11, D12)
4. تست LED در setup را اجرا کنید (باید در بوت از میان رنگ‌ها بچرخد)

---

### مشکل: بازر صدا نمی‌دهد

**علائم:**
- هنگام فعال‌شدن سیستم صدایی نیست

**راه‌حل‌ها:**
1. تأیید کنید بازر از نوع **فعال** است (نه غیرفعال)
2. قطبیت را بررسی کنید (پایه مثبت به D5)
3. با اسکچ ساده تست کنید:
   ```cpp
   void setup() {
     pinMode(5, OUTPUT);
   }
   void loop() {
     digitalWrite(5, HIGH);
     delay(1000);
     digitalWrite(5, LOW);
     delay(1000);
   }
   ```

---

### مشکل: سنسور هال تشخیص نمی‌دهد

**علائم:**
- مقدار هال با آهنربا تغییر نمی‌کند

**راه‌حل‌ها:**
1. تأیید کنید سنسور از نوع **آنالوگ** است (نه دیجیتال)
2. سیم‌کشی را بررسی کنید: VCC→5V, GND→GND, OUT→A0
3. با مولتی‌متر تست کنید (باید 0-5V بخواند)
4. آهنربای متفاوتی امتحان کنید (نئودیمیوم قوی‌تر توصیه می‌شود)
5. Serial Monitor را برای مقادیر خام بررسی کنید

---

### مشکل: LDR پاسخ نمی‌دهد

**علائم:**
- مقدار LDR ثابت می‌ماند

**راه‌حل‌ها:**
1. مدار تقسیم‌کننده ولتاژ را تأیید کنید:
   ```
   5V → LDR → A1 → 10KΩ → GND
   ```
2. مقدار مقاومت 10KΩ را بررسی کنید
3. با مولتی‌متر تست کنید (مقاومت باید با نور تغییر کند)
4. LDR متفاوتی امتحان کنید (GL5528 توصیه می‌شود)

---

### مشکل: سنسور لمسی کار نمی‌کند

**علائم:**
- لمس رویدادها را فعال نمی‌کند

**راه‌حل‌ها:**
1. سیم‌کشی را بررسی کنید: VCC→5V, GND→GND, OUT→D4
2. نوع سنسور را تأیید کنید (TTP223 یا مشابه)
3. با اسکچ ساده تست کنید:
   ```cpp
   void setup() {
     Serial.begin(9600);
     pinMode(4, INPUT);
   }
   void loop() {
     Serial.println(digitalRead(4));
     delay(100);
   }
   ```
4. حساسیت لمس را تنظیم کنید (اگر ماژول پتانسیومتر دارد)

---

### مشکل: WiFi متصل نمی‌شود

**علائم:**
- ESP8266 در Serial Monitor "WiFi: Failed" را نشان می‌دهد

**راه‌حل‌ها:**
1. SSID و رمز عبور را در کد تأیید کنید (حساس به حروف بزرگ/کوچک!)
2. بررسی کنید WiFi 2.4GHz است (ESP8266 از 5GHz پشتیبانی نمی‌کند)
3. ESP را به روتر نزدیک‌تر ببرید
4. فیلتر MAC روتر را بررسی کنید (آدرس MAC ESP را اضافه کنید)
5. شبکه WiFi متفاوتی امتحان کنید

---

### مشکل: داشبورد "آفلاین" را نشان می‌دهد

**علائم:**
- داشبورد وب دستگاه را آفلاین نشان می‌دهد

**راه‌حل‌ها:**
1. تأیید کنید IP سرور در کد ESP با IP واقعی سرور مطابقت دارد
2. تنظیمات فایروال را بررسی کنید (اجازه پورت 5000)
3. مطمئن شوید سرور در حال اجرا است (`python app.py`)
4. Serial Monitor ESP را برای خطاهای HTTP بررسی کنید
5. تأیید کنید هر دو دستگاه در شبکه یکسان هستند

---

### مشکل: رنگ‌های LED اشتباه

**علائم:**
- قرمز به عنوان آبی نشان داده می‌شود، سبز به عنوان زرد، و غیره

**راه‌حل‌ها:**
1. بررسی کنید تخصیص‌های پین در کد با سیم‌کشی فیزیکی مطابقت دارند
2. تأیید کنید رنگ‌های LED با برچسب‌های پین مطابقت دارند
3. در صورت نیاز اعداد پین را در کد جابجا کنید:
   ```cpp
   const int PIN_RED = 11;    // با آبی جابجا کنید
   const int PIN_BLUE = 9;    // با قرمز جابجا کنید
   ```

---

### مشکل: هشدارهای کاذب

**علائم:**
- سیستم بدون نفوذ واقعی فعال می‌شود

**راه‌حل‌ها:**
1. آستانه‌های هیسترزیس را در کد ESP تنظیم کنید:
   ```cpp
   #define HALL_DANGER_MIN 460  // برای باریک‌کردن منطقه خطر افزایش دهید
   #define HALL_DANGER_MAX 540  // برای باریک‌کردن منطقه خطر کاهش دهید
   ```
2. تداخل الکترومغناطیسی را بررسی کنید (موتورها، ترانسفورماتورها)
3. تأیید کنید آهنربا محکم نصب شده است
4. مقدار `ALARM_COOLDOWN` را افزایش دهید
5. LDR را برای تغییرات ناگهانی نور بررسی کنید (سایه‌ها، ابرها)

---

### مشکل: سیستم در حالت فعال‌شده گیر کرده است

**علائم:**
- سیستم از حالت هشدار خارج نمی‌شود

**راه‌حل‌ها:**
1. از داشبورد وب برای غیرفعال‌سازی استفاده کنید (رمز عبور را وارد کنید)
2. سنسور لمسی را به طور کوتاه فشار دهید تا بازنشانی شود
3. سیستم را خاموش/روشن کنید
4. Serial Monitor را برای پیام‌های خطا بررسی کنید
5. تأیید کنید `syncStatus()` در کد ESP کار می‌کند

---

### مشکل: خطاهای ارتباط سریال

**علائم:**
- داده‌های خراب بین ATmega و ESP

**راه‌حل‌ها:**
1. تأیید کنید نرخ baud مطابقت دارد (9600 در هر دو اسکچ)
2. اتصالات TX/RX را بررسی کنید (آنها را متقاطع کنید: TX→RX, RX→TX)
3. اگر ESP 3.3V و ATmega 5V است، تقسیم‌کننده ولتاژ اضافه کنید:
   ```
   ATmega TX → 1KΩ → ESP RX → 2KΩ → GND
   ```
4. از SoftwareSerial برای دیباگ استفاده کنید
5. حلقه‌های زمین را بررسی کنید (تمام GNDها را به هم متصل کنید)

## 🔒 ملاحظات امنیتی

### اقدامات امنیتی فعلی
- محافظت با رمز عبور برای داشبورد وب
- فقط شبکه محلی (در اینترنت قرار نگرفته)
- هیچ داده حساسی به صورت متن ساده ذخیره نمی‌شود
- ثبت رویدادها برای ردیابی حسابرسی

### بهبودهای توصیه‌شده
1. **HTTPS**: استفاده از SSL/TLS برای ارتباط رمزنگاری‌شده
2. **رمزهای عبور قوی**: الزامات رمز عبور پیچیده را اعمال کنید
3. **محدودیت نرخ**: جلوگیری از حملات brute-force بر ورود
4. **مدیریت نشست**: پیاده‌سازی توکن‌های نشست امن
5. **اعتبارسنجی ورودی**: تمام ورودی‌های کاربر را پاکسازی کنید
6. **بخش‌بندی شبکه**: دستگاه‌های IoT را در VLAN جداگانه قرار دهید
7. **به‌روزرسانی‌های منظم**: تمام اجزای نرم‌افزاری را به‌روز نگه دارید
8. **احراز هویت دو عاملی**: افزودن 2FA برای دسترسی وب

### محدودیت‌های شناخته‌شده
- رمز عبور پیش‌فرض (0000) باید بلافاصله تغییر کند
- هیچ رمزنگاری برای ارتباط سریال بین میکروکنترلرها وجود ندارد
- اعتبارنامه‌های WiFi به صورت متن ساده در فریم‌ور ذخیره شده‌اند
- هیچ تشخیص نفوذی برای خود سیستم وجود ندارد

4. به شاخه push کنید (`git push origin feature/AmazingFeature`)
5. یک Pull Request باز کنید

### دستورالعمل‌های توسعه
- از سبک کد موجود پیروی کنید
- برای منطق پیچیده کامنت اضافه کنید
- مستندات را برای ویژگی‌های جدید به‌روزرسانی کنید
- قبل از ارسال به طور کامل تست کنید
- برای تغییرات UI تصاویر را شامل کنید

### سبک کد
- **C/C++**: از راهنمای سبک Arduino پیروی کنید
- **Python**: از PEP 8 پیروی کنید
- **HTML/CSS/JS**: از الگوهای موجود پیروی کنید
- **پیام‌های Commit**: از قالب conventional commits استفاده کنید
